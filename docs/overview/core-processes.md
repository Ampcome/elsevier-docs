# Core Publishing Services Processes

## Overview

Publishing Services handles 12 different processes across the end-to-end publishing workflow. These processes support multiple business units in different geographies.

## The 12 Core Processes

| Process Category | Process Name | Description |
|------------------|--------------|-------------|
| **Setup** | EMSS Site Set-up | Configure manuscript submission environment |
| **Contributors** | List of Contributors Confirmation | Verify and confirm all contributors |
| **Review** | Proposal Review Administration | Manage proposal review workflow |
| **Copies** | Gratis Copies Management | Handle complimentary copy distribution |
| **Data** | Data Management | Maintain data integrity across systems |
| **Assessment** | Technical Assessment | Evaluate technical quality |
| **Agreements** | Agreements Management | Handle author/contributor contracts |
| **Payee** | Payee Information Management | Manage payment recipient data |
| **Permissions** | Permissions Seeking | Obtain rights from external holders |
| **Permissions** | Permissions Granting | Process incoming permission requests |
| **Corrections** | Post-Publication Corrections | Handle corrections after publication |
| **CRM** | Customer Relationship Management | Manage stakeholder relationships |

## Supporting Processes

The following processes run alongside the main workflow:

- **Agreements Management** - Contract handling throughout project lifecycle
- **Permissions Seeking** - Proactive rights acquisition
- **LOC Administration** - List of Contributors management
- **Technical Assessment** - Quality evaluation
- **Data Management** - System data synchronization
- **Payee Information Management** - Payment processing support
- **Customer Relationship Management** - Stakeholder communication
